## 1. 콘텐츠 운영 원칙

### 발행 체계

- **AI NEWS:** 매일 2개 발행
    - **Research 포스트:** 기술 심화 (자동 발행) — 모델, 논문, 오픈소스 중심. 뉴스 없는 날은 "없음" 공지 + 최근 동향 보충
    - **Business 포스트:** 시장 분석 (수동 검수) — 2-step call flow (Call 1: Expert EN+KO, Call 2: Learner EN+KO) + Related News 3카테고리. 작성자가 검수 후 발행
- **서재 (Study):** 주 1~2회 — 깊이 있는 기술 콘텐츠, 코드 포함
- **커리어 (Career):** 비정기 — 작성자의 진솔한 목소리 우선
- **프로젝트 (Project):** 비정기 — 실제 개발 과정 기록

### AI 워크플로우 분류

| 타입 | 적용 카테고리 | 방식 |
| --- | --- | --- |
| **Research 
(자동 발행)** | AI NEWS 
(기술 심화) | Tavily 수집 → 랭킹 → 단일 기술 심화 포스트 생성 → 자동 발행 |
| **Business 
(수동 검수)** | AI NEWS 
(시장 분석) | Tavily 수집 → 랭킹 → 3페르소나 포스트 + Related News → 수동 검수 발행 |
| **Type A: 
Multi-Targeting** | 학습 | 하나의 원문 → 3개 페르소나 버전 자동 생성 |
| **Type B: 
Authentic Voice** | 커리어, 프로젝트 | 작성자 목소리 유지, AI는 구성/문장 다듬기 보조 |

### 다국어 운영 계약 (07 연동)

| 항목 | 계약 |
|---|---|
| **원본 언어** | EN을 canonical source로 사용 |
| **파생 언어** | KO는 EN 기준 localized derivative로 생성 |
| **발행 기본 방향** | EN-first 운영, KO는 로컬라이징 검수 후 발행 |
| **독자 가치** | EN: 고밀도 큐레이션 / KO: 정보 비대칭 해소 + 맥락화 |
| **금지 사항** | 직역 중심 기계 번역투 콘텐츠, 근거 없는 과장형 카피 |

## 2. 콘텐츠 가치 훅 (Core Value Hooks)

단순 정보 전달이 아닌, "오직 여기서만 얻을 수 있는 가치"를 만드는 4가지 전략적 장치.

### ① "So What?" — 실천적 가치 제안

뉴스를 읽고 난 후, 독자가 당장 무엇을 해야 할지 정의한다.

- **입문자:** "오늘 대화에서 아는 척하기 좋은 한 문장 요약"
- **학습자:** "오늘 당장 내 프로젝트에 적용해 볼 수 있는 라이브러리/함수"
- **현직자:** "내일 회의에서 제안해 볼 만한 비즈니스 아이디어"

### ② "Connecting the Dots" — 맥락적 연결

단발성 소식이 아닌, 과거 사건이나 타 기술과의 연결 고리를 'PM의 시선'으로 분석한다.

- 관련 과거 포스팅 링크를 AI가 자동 추천
- 두 소식 사이의 인과관계/경쟁 구도 분석
- 예시: "지난주 OpenAI 발표와 이번 Google의 대응, 사실은 이 지점에서 만납니다."

### ③ "The Prediction Game" — 예측과 베팅

뉴스 기반 미래 예측 + 게이미피케이션으로 재방문 유도. (Phase 4에서 본격 구현)

- 입문자용 미니 퀴즈 (뉴스 복습)
- 트렌드 베팅 및 투표 시스템
- 예측 성공 시 포인트 보상

### ④ "PM's Cost & Strategy Radar" — 비즈니스 리얼리티

엔지니어링 블로그가 놓치기 쉬운 '돈과 전략' 관점.

- 새 모델/서비스 출시 시 기존 대안 대비 가성비 비교 차트
- 예시: "성능은 2배 좋아졌지만, API 비용은 3배 비쌉니다. 쓸 가치가 있을까요?"

## 3. 프롬프트 가이드 — 전략적 추론 항목 (v1.3)

AI 콘텐츠 생성 시 아래 항목을 활용한다. 매일 모든 항목을 채우는 것이 아니라, 디폴트 + 회전 구조로 운영하여 콘텐츠의 가독성과 발행 지속가능성을 확보한다.

### 전체 항목 리스트

| # | 항목 | 설명 | 용도 |
| --- | --- | --- | --- |
| 1 | **[The One-Liner]** | 핵심 정의 한 문장 (필요시 비유 포함) | 디폴트 (입문자) |
| 2 | **[Action Item]** | 당장 적용할 수 있는 것 (Dev/PM 구분) | 디폴트 (학습자) |
| 3 | **[Critical Gotcha]** | 화려한 수치 뒤 한계점 리얼리티 체크 | 디폴트 (현직자) |
| 4 | **[Market Context]** | Winner & Loser + Why Now 통합 시장 맥락 분석 | 회전 |
| 5 | **[Analogy for Laypeople]** | 일상 비유 설명 (기술 장벽이 높은 뉴스에 적합) | 회전 |
| 6 | **[Source Check]** | 출처 유형 + 정보 신뢰도 한 줄 표기 | 회전 |
| + | **[Today's Quiz/Poll]** | 뉴스 기반 퀴즈 또는 투표 주제 | 고정 (하단) |

> **참고:** [Today's Quiz/Poll]은 Phase 4(포인트 시스템) 구현 전까지 UI 인터랙션 없이 텍스트로만 노출.
> 

### 매일 AI NEWS 포스트 구성 (5블록 구조)

| 슬롯 | 항목 | 비고 |
| --- | --- | --- |
| 디폴트 (입문자) | [The One-Liner] | 항상 노출 |
| 디폴트 (학습자) | [Action Item] | 항상 노출 |
| 디폴트 (현직자) | [Critical Gotcha] | 항상 노출 |
| 회전 항목 (1개) | [Market Context] / [Analogy] / [Source Check] 중 택 1 | 뉴스 성격에 맞게 선택 |
| 고정 (하단) | [Today's Quiz/Poll] | Phase 4 전까지 텍스트만 |

> **운영 원칙:** 5블록 구조는 AI NEWS Business 포스트에 적용된다. Research 포스트는 단일 기술 심화 버전으로 Critical Gotcha + Action Item 중심의 구성을 따른다. 디폴트 3개는 각 페르소나 포스트에 기본 포함되며, 회전 항목은 해당 뉴스에 가장 적합한 1개를 선택하여 추가한다. EN/KO 모두 동일한 사실 집합을 기준으로 작성한다.
> 

### Locale별 5블록 생성/노출 계약 (Contract)

| 구분 | EN (Canonical) | KO (Localized) |
|---|---|---|
| **저장 규칙** | `guide_items` 5블록 전체 생성/저장 | EN 기준 의미 보존 + 5블록 전체 생성/저장 |
| **기본 노출 (Tier A)** | 5블록 전체 노출 | 5블록 전체 노출 |
| **기본 노출 (Tier B)** | Compact 노출 허용(핵심 3블록 우선), 단 저장은 5블록 유지 | 5블록 노출 유지(요약 강도 조절 가능) |
| **Quiz/Poll 처리** | 텍스트 또는 링크형 안내 | 텍스트 또는 링크형 안내 |
| **검수 포인트** | 증거/한계/맥락 명확성 | 자연스러운 한국어/국내 맥락 전달 |

> **핵심 계약:** EN이 요약형으로 노출되더라도, 데이터 저장 스키마는 KO/EN 모두 5블록을 유지한다.

## 4. 페르소나 시스템

### 2가지 독자 페르소나

**2026-03-17 변경사항:** Beginner 페르소나는 제거되었습니다. 모든 콘텐츠가 Learner + Expert 2가지 쓰기 스타일로 통일되며, 동일한 정보량(5,000-7,000자)을 제공합니다. Beginner를 위한 비유와 배경지식 설명은 Learner 버전에 통합되었습니다.

| 페르소나 | 톤 & 스타일 | 디폴트 항목 |
| --- | --- | --- |
| **학습자** | 핵심 개념 + 관련 학습 링크 + 왜 중요한지 맥락, 비유와 배경지식 포함 | #2 [Action Item] |
| **현직자** | 기술적 세부사항 + 업계 영향도 + 실무 적용, 전략적 인사이트 | #3 [Critical Gotcha] |

### Locale × Persona 매핑 계약

| 축 | KO | EN |
|---|---|---|
| **페르소나 라벨** | 학습자 / 현직자 | Learner / Expert |
| **의미 매핑** | 동일 | 동일 |
| **톤 기준** | 친절하고 자연스러운 설명 + 한국 시장 맥락 | 간결하고 증거 중심 + 과장 금지 |
| **분량 기준** | 모든 페르소나 동일 (5,000-7,000자), 서술 방식으로만 차별화 | 모든 페르소나 동일 (5,000-7,000자), 서술 방식으로만 차별화 |
| **콘텐츠 차이 허용 범위** | 비유/예시의 현지화 허용 | 요약 밀도 최적화 허용 |
| **불변 조건** | 사실/수치/출처는 EN canonical과 일치 | 사실/수치/출처가 기준 원본 |

### 페르소나 전환 UX

- **적용 범위:** AI NEWS(Business 포스트), 서재(Study) 카테고리에 한해 Persona Switcher 활성화. Research 포스트는 단일 버전이므로 Switcher 없음
- **기본값 우선순위:** 로그인 사용자 `DB > 쿠키 > beginner`, 비로그인 사용자는 쿠키 우선
- **전환:** 각 포스트 상단 탭에서 언제든 다른 페르소나로 자유 전환 가능
- **비로그인:** 쿠키/로컬스토리지로 선택 기억
- **로그인(Phase 4):** 프로필에 저장, 기기 간 동기화
- **언어 전환:** 언어 스위처 사용 시 동일 `translation_group_id`의 KO/EN 포스트로 이동

### Locale별 톤 & 스타일 가드레일

| 구분 | EN 작성 가이드 | KO 작성 가이드 |
|---|---|---|
| **톤** | concise, evidence-first, no hype | 친절/명료, 번역투 금지 |
| **문장 운영** | 짧은 문장, 신호 중심 압축 | 문맥 연결 강화, 독자 이해 우선 |
| **리스크 표기** | 제한점/불확실성 명시 필수 | 한계/주의점 한국어로 명확화 |
| **금지** | 과장형 CTA, 근거 없는 단정 | 직역체, 영어 문장 구조 직수입 |
| **공통 필수** | 수치·비용·성능 주장에 출처 링크 포함 | 수치·비용·성능 주장에 출처 링크 포함 |

## 5. 시각적 요소

### 뉴스 '온도' 시각화

- 매우 혁신적인 뉴스: Neon Pink 강조 + 불꽃 아이콘
- 일반 뉴스: 기본 스타일

### 타임라인 인디케이터 (Phase 3)

- 해당 기술의 역사(History) 중 어디쯤에 와 있는지 보여주는 프로그레스 바
- 예: "LLM 추론 최적화" 기술이 초기 연구 → 상용화 중 어느 단계인지 표시

### Highlight to Share (Phase 3)

- 본문 문장 드래그 시 X(Twitter) / LinkedIn 공유 버튼 팝업
- 해당 문장 + 블로그 링크 포함 카드 자동 생성
- 공유 시 20pt 지급 (Phase 4 포인트 시스템 연동)

## 9. 다국어 콘텐츠 계약 — 성공 기준 (Acceptance Criteria)

1. EN/KO 모두 동일 사실 집합(출처/수치/핵심 주장) 기반으로 작성된다.
2. EN은 canonical source로 유지되고, KO는 localized derivative로 일관되게 표현된다.
3. Business 포스트는 EN/KO 모두 `guide_items` 5블록이 저장된다.
4. Tier B에서 EN compact 노출을 적용해도 저장 스키마(5블록)는 유지된다.
5. Persona 기본값은 `DB > 쿠키 > beginner` 규칙과 충돌하지 않는다.
6. Locale 전환 시 `translation_group_id` 기준으로 같은 콘텐츠 페어로 이동한다.
7. EN 문체는 evidence-first, KO 문체는 자연스러운 한국어 설명 중심을 충족한다.
