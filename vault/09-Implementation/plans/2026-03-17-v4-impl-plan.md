# News Pipeline v4 (2 Persona) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 3 페르소나(Expert/Learner/Beginner) → 2 페르소나(Expert 현직자 / Learner 학습자) 전환. Beginner 제거, Learner에 통합.

**Architecture:** DB 컬럼은 유지(backward compat), 코드에서 Beginner 생성/표시를 제거. 기존 3-탭 포스트는 자연스럽게 2-탭으로 전환(content_beginner 존재 여부로 판단).

**Tech Stack:** FastAPI (Python), Astro v5 + Tailwind CSS v4, Supabase (PostgreSQL), OpenAI gpt-4o

### Task 5: Frontend — newsDetailPage.ts 수정

**Files:**
- Modify: `frontend/src/lib/pageData/newsDetailPage.ts`

**Step 1: contentMap에서 beginner 처리 변경**

```typescript
// Before
const contentMap: Record<string, string> = {
  beginner: post.content_beginner || '',
  learner: post.content_learner || '',
  expert: post.content_expert || '',
};
rawContent = contentMap[personaKey] || post.content_learner || post.content_beginner || '';

// After — backward compat: beginner 키는 유지하되 fallback 순서 변경

  learner: post.content_learner || '',
  expert: post.content_expert || '',
  ...(post.content_beginner ? { beginner: post.content_beginner } : {}),
};
rawContent = contentMap[personaKey] || post.content_learner || '';
activePersona = contentMap[personaKey] ? personaKey : 'learner';
```

핵심: 기존 포스트에 `content_beginner`가 있으면 3탭 유지, 없으면 2탭.
