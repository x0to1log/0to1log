# README Design — 2026-03-26

> **목표:** 프로젝트를 처음 보는 모든 사람(사용자, 개발자, 파트너)을 위한 명확하고 설득력 있는 README 작성
> **포지셔닝:** "가치(왜) + 기술(어떻게)" 균형, 문제 정의로 입구
> **상태:** Design Approved (2026-03-26)

---

## 설계 원칙

1. **문제 정의부터 시작** — 비주얼 자산이 없으므로 "왜 필요한가"로 자연스럽게 가치 드러내기
2. **기술 깊이 유지 + 모두 이해 가능** — 처음 소개할 땐 쉽게, 그 후로는 기술 용어 그대로 사용
3. **Stable Core + Ongoing Improvements** — 현재 상태를 명확히 (사용 가능하되, 진행 중인 부분도 명시)
4. **다중 대상, 명확한 섹션** — 개발자/사용자/관심자가 각자 필요한 정보를 빠르게 찾을 수 있게
5. **향후 프라이빗 전환 가능** — 기여 가이드는 가볍게 ("관심 있으면 연락") 정도로만

## 섹션 구조 (7개)

### 1️⃣ 짧은 소개 (한 줄)
```
한국인을 위한 AI 뉴스 큐레이션 + 용어집 + IT 블로그 플랫폼
```

### 2️⃣ 문제 정의 (왜 필요한가?)

**맥락:**
- AI 뉴스는 영어가 제일 빠르고 다양
- 한국어만 접하면 정보 격차

**네 가지 문제:**
1. 뉴스가 너무 많다 → 정말 중요한 건 무엇인가?
2. 영문 기술 용어가 어렵다 → "Agent", "RAG"... 정확히 뭐고 왜 중요한가?
3. 뉴스와 학습이 분리돼 있다 → 오늘의 뉴스 속 개념을 체계적으로 배우고 싶다면?
4. 배운 후 도구를 못 찾는다 → AI 제품이 너무 많은데, 어디서 찾고 어떤 게 나한테 맞는가?

**결론:** "0to1log는 이 네 가지를 한 곳에서 해결한다. 뉴스, 학습, 도구 탐색이 분리되지 않는, 최초의 AI 학습 플랫폼."

**톤:** 객관적이고 설득력 있게, 비난하지 않고 현실적으로.

---

### 3️⃣ 솔루션 (0to1log의 세 기둥)

#### 🔍 Daily News Digest
- **기능:** 영어 AI 뉴스를 LLM으로 자동 큐레이션
- **특징:**
  - Research 페르소나 (논문·기술 트렌드)
  - Business 페르소나 (비즈니스·업계 영향)
  - 한국어 + 영어 동시 제공
- **해결하는 것:** 문제 1, 2

#### 📚 AI Handbook
- **기능:** 핵심 AI 용어를 체계적으로 설명
- **특징:**
  - 기초: 입문자용
  - 심화: 개발자/실무자용
  - 뉴스 속 용어 자동 인라인 연결
- **해결하는 것:** 문제 2, 3

#### ✍️ Blog & AI Product Guides
- **기능:** AI 도구 선택 가이드 + IT 인사이트 글
- **특징:**
  - 제품 비교 가이드
  - 사용 사례
  - 심화 기술 블로그
- **해결하는 것:** 문제 3, 4

**톤:** 각 기능을 간결하게, 왜 필요한지 명시.

### 4️⃣ 기술 스택

**구조:** 레이어별 도구 + 한 줄 설명 + 링크

- **Frontend:** Astro v5, Tailwind CSS v4, Vercel
- **Backend:** FastAPI, PydanticAI, Railway
- **Database:** Supabase (PostgreSQL + Auth + RLS)
- **AI & Search:** OpenAI (gpt-4.1, gpt-4.1-mini), Tavily API
- **Automation:** Cron jobs, GitHub Actions

**톤:** 기술 선택이 현대적이고 견고하다는 느낌.

## 작성 가이드

- **길이:** 각 섹션 50~150 단어 (너무 길지 않게)
- **마크다운 구조:** 제목 + 불릿 + 설명
- **링크:** 내부 링크는 상대 경로 (vault/, frontend/, backend/)
- **톤:** 친근하되 전문적, 과장하지 않기
- **AI 용어:** 처음 등장할 때는 간단히 설명, 그 후로는 그대로 사용
