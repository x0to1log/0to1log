## v1 콘텐츠 구성

포스트 1건에 포함되는 요소:

1. **3 페르소나 본문** — 탭 UI로 전환
2. **출처** — 원본 소스 + 커뮤니티 반응 (출처 링크 포함)
3. **핵심 요약 (TL;DR)** — 3줄 이내, 리스트/카드 미리보기용
4. **Handbook 용어 링크** — 본문 내 AI 용어를 용어집으로 연결

## 페르소나별 고정 섹션

### 현직자 (Expert) — "이걸로 뭘 해야 하지?"

| 섹션 | 역할 |
|------|------|
| **## 핵심 요약** | 3줄 executive summary. 바쁜 의사결정자가 여기만 읽어도 핵심 파악 |
| **## 기술 심층 분석** | 아키텍처, 벤치마크, 기존 대비 차이. 구체적 수치 필수 |
| **## 시장 & 경쟁 분석** | 누가 이기고 지는지, 투자/파트너십 시그널, 규제 변화 |
| **## 전략적 시사점** | build/buy/wait 판단, 마이그레이션 경로, 6개월 전망 |

### 학습자 (Learner) — "이해하고 적용하고 싶어"

| 섹션 | 역할 |
|------|------|
| **## 무슨 일이 있었나** | 누가, 뭘, 왜. 배경 맥락과 함께 팩트 전달 |
| **## 기술 해부** | 어떻게 작동하는지. 코드 스니펫, 비교 테이블 활용 가능 |
| **## 실무 적용** | 개발자/PM 관점에서 "내 업무에 어떤 영향?" |
| **## 참고 자료** | 공식 문서, 튜토리얼, GitHub 레포 링크 |

### 입문자 (Beginner) — "왜 이게 중요한 거야?"

| 섹션 | 역할 |
|------|------|
| **## 한 줄 요약** | 이 뉴스를 한 문장으로 |
| **## 무슨 일이 있었나** | 배경지식 없이도 이해할 수 있는 설명. Handbook 링크 적극 활용 |
| **## 왜 중요한가** | 일상/사회/직업에 미치는 영향 |
| **## 알아두면 좋은 것** | 이 주제를 더 이해하기 위한 기본 개념 정리. Handbook 연계 |

### 공통 필수 규칙
- 모든 주장에 **출처 인라인** (`[출처명](URL)`)
- AI 용어에 **Handbook 링크** 연결

## 이중 언어 전략: EN+KO 동시 생성

- ~~EN 생성 → KO 번역 (v1에서 폐기)~~
