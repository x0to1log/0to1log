## v5 콘텐츠 구성

포스트 1건에 포함되는 요소:

1. **2 페르소나 본문** — 탭 UI로 전환 (Expert / Learner)
2. **출처** — 원본 소스 카드 + 커뮤니티 반응 (Reddit/HN)
3. **핵심 요약 (TL;DR)** — excerpt, 리스트/카드 미리보기용
4. **Handbook 용어 링크** — 본문 내 AI 용어를 용어집으로 연결
5. **퀴즈** — 페르소나별 quiz_en + quiz_ko
6. **가이드 블록** — 5블록 구조 (guide_items)
7. **관련 태그 / 포커스 아이템** — tags, focus_items 자동 생성

## 페르소나별 고정 섹션

### 현직자 (Expert) — "이걸로 뭘 해야 하지?"

| 섹션 | 역할 |
|---|---|
| **## 핵심 요약** | 3줄 executive summary. 바쁜 의사결정자가 여기만 읽어도 핵심 파악 |
| **## 기술 심층 분석** | 아키텍처, 벤치마크, 기존 대비 차이. 구체적 수치 필수 |
| **## 시장 & 경쟁 분석** | 누가 이기고 지는지, 투자/파트너십 시그널, 규제 변화 |
| **## 전략적 시사점** | build/buy/wait 판단, 마이그레이션 경로, 6개월 전망 |

### 학습자 (Learner) — "이해하고 적용하고 싶어"

| 섹션 | 역할 |
|---|---|
| **## 무슨 일이 있었나** | 누가, 뭘, 왜. 배경 맥락과 함께 팩트 전달 |
| **## 기술 해부** | 어떻게 작동하는지. 코드 스니펫, 비교 테이블 활용 가능 |
| **## 실무 적용** | 개발자/PM 관점에서 "내 업무에 어떤 영향?" |
| **## 참고 자료** | 공식 문서, 튜토리얼, GitHub 레포 링크 |

### 공통 필수 규칙
- 모든 주장에 **출처 인라인** (`[출처명](URL)`)
- AI 용어에 **Handbook 링크** 연결
- ==두 페르소나 동일 분량== (목표 6,000~7,000자), 서술 방식만 차별화

## 이중 언어 전략: EN+KO 동시 생성

- ~~EN 생성 → KO 번역 (v1에서 폐기)~~
- **페르소나 호출마다 EN/KO 두 버전을 JSON으로 함께 출력**
- 번역 단계 완전 제거 → 번역체(번역투) 문제 해결
- 한국어가 처음부터 자연스럽게 작성됨
- `max_tokens=16,000` (EN + KO + JSON 오버헤드 충분)
- EN/KO 별도 row로 저장 (`translation_group_id`로 쌍 연결)
