# 2026-03-18 — 핸드북 심화 퀄리티 시스템 설계

## 배경

핸드북 심화(Advanced) 콘텐츠가 "기술 블로그" 수준에 머물러 있어서 시니어 엔지니어가 참고할 수준이 안 됨.

## 발견한 문제

1. **모든 용어에 동일한 프롬프트** → "수식 필수"인데 비즈니스 용어에는 수식이 해당 없음
2. **LLM knowledge cutoff** → 최신 제품(GPT-5.4 등)에 대한 정확한 정보 없음
3. **퀄리티 검증 없음** → 얕은 콘텐츠가 그대로 저장됨
4. **One-shot 생성** → 자기 검토 없이 한 번에 끝

## 핵심 결정

### 10개 용어 유형 분류 체계
- 알고리즘/모델, 인프라/도구, 비즈니스/산업, 개념/이론, 제품/브랜드 (기존 5개)
- **+ 메트릭/지표, 기법/방법론, 데이터 구조/포맷, 프로토콜/표준, 아키텍처 패턴** (신규 5개)
- 각 유형마다 "깊이의 축"이 완전히 다름 → 유형별 전용 프롬프트 필요

### gpt-4o-mini로 유형 분류
- 10개 유형 전부를 심화 프롬프트에 넣으면 LLM이 소화 못 함
- 분류 호출을 분리해서 해당 유형 프롬프트만 전달 → 짧고 집중적
- 비용: ~$0.001/용어

### Tavily 검색 필수 적용
- 모든 용어에 Tavily 검색 결과를 context로 전달
- 최신 정보, 벤치마크 수치, 실제 URL 확보
- 비용: 월간 크레딧으로 무료

### Self-critique + 퀄리티 체크
- 생성 후 LLM이 자기 출력을 시니어 엔지니어 관점으로 검토
- 부족하면 1회 보강 재생성
- 퀄리티 점수 매기기 (0-100)

## 비용 영향

+$0.07/용어 (22% 증가), 월 +$10.50

## 교훈

- **프롬프트만으로는 한계가 있다** — LLM이 모르는 건 지시해도 못 씀
- **입력(context)을 풍부하게 하는 게 프롬프트 지시 강화보다 효과적**
- 뉴스 다이제스트의 `article_context` 패턴이 핸드북에도 동일하게 적용됨
- **유형별 분리가 핵심** — 하나의 프롬프트로 모든 유형을 커버하려는 건 한계
