## 요약

핸드북 용어집의 콘텐츠 구조를 전면 개편. Basic/Advanced 모두에 새 섹션 추가, 팝업 최적화, 수식 렌더링 분리, 파이프라인 안정화까지 하루에 완료.

## 변경 내역

### 1. Basic 콘텐츠 — 3개 섹션 추가

| 섹션 | 목적 | 설계 의도 |
|------|------|-----------|
| **30초 요약** (0_summary) | 팝업 + 상세 페이지 맨 위 | 전문 용어 0, 비유 기반. "문제→비유→한계→결론" 구조 |
| **직군별 활용 포인트** (9_roles) | "나한테 왜 중요한가" | 주니어/PM/시니어/비개발직군별 구체적 액션 |
| **다음에 읽을 것** (10_learning_path) | 재방문 + 체류시간 | 학습 순서 3개 용어 연결 |

### 2. Advanced 콘텐츠 — 2개 섹션 추가

| 섹션 | 목적 |
|------|------|
| **언제 써야 하나** (10_when_to_use) | 적합/부적합 시나리오 + 대안 기술 명시 |
| **흔한 실수와 해결** (11_pitfalls) | 실무 실수-해결 쌍 3~4개 |

### 3. 30초 요약 설계 — 팝업과 상세 페이지 동시 서빙

**두 가지 소비 경로를 하나의 필드로 해결:**
- 뉴스 팝업 (3초): `extractFirstSection`이 30초 요약 전체를 표시
- 상세 페이지 (10초): 맨 위에 요약 카드로 표시

**프롬프트 구조:**
```
[문제/상황] AI에게 질문하면 가끔 틀린 답을 자신 있게 말한다.
[비유] RAG는 이 문제를 해결하기 위해 AI가 답하기 전에 관련 자료를
       먼저 검색해서 확인하는 기술이다.
[한계] 다만 검색이 잘못되면 답도 틀릴 수 있다.
[결론] -> 요즘 출시되는 AI 챗봇 대부분이 이 방식을 쓴다.
```

핵심 규칙: **전문 용어 사용 절대 금지** — "LLM", "embedding", "hallucination" 등 banned. 이 용어를 모르는 사람이 클릭했으니까.

5개 유형별 예시 제공: 개념(RAG, Transformer), 모델(GPT-5.4), 도구(LangChain), 지표(F1 Score)

### 4. 현상/문제 유형 대응

Hallucination, Overfitting 같은 "기술이 아닌 현상" 용어에 대해:
- "어디서 쓰이나" → "어디서 발생하나"
- "직군별 활용" → "직군별 대응"
- "언제 써야 하나" → "언제 주의해야 하나"

섹션 키는 동일(조립 코드 변경 없음), 콘텐츠 관점만 LLM이 자동 조정.

### 5. 섹션 순서 최적화

**Basic:**
```
30초 요약 → 쉽게 이해 → 예시 → 한눈에 보기 → 왜 중요 →
어디서 쓰이나 → 직군별 → 주의할 점 → 대화 → 관련 용어 → 다음에 읽을 것
```

**Advanced:**
```
기술 설명 → 수식 → 동작 원리 → 코드 → 실무 →
언제 써야 → 흔한 실수 → 왜 중요 → 대화 → 참조 → 관련 기술
```

아코디언 경계: 5번째 → 6번째 h2로 이동 (0_summary 추가로 인한 밀림 보정)

### 6. 수식 렌더링 — 핸드북 전용 프로세서

**문제:** 뉴스는 `$2` (달러), 핸드북은 `$x^2$` (수식). 같은 `$`인데 의미가 다름.

**해결:** `handbookProcessor` 분리
- 뉴스: `singleDollarTextMath: false` (달러 보호)
- 핸드북 Advanced: `singleDollarTextMath: true` (수식 렌더링)

추가: auto-convert 후처리 — LLM이 `$...$`로 출력해도 저장 전에 `$$...$$`로 변환

### 7. 용어 하이라이트 규칙 정리

| 위치 | 규칙 |
|------|------|
| 본문 | first-occurrence only |
| 관련 기술 & 비교 | 각 용어 1번씩 (본문과 독립 추적) |

### 8. 파이프라인 안정화

- Generate All Fields: 동기 → background task (Vercel 60초 타임아웃 회피)
- background task 완료 시 DB 자동 저장 (polling 실패해도 데이터 보존)
- 3단계 타임아웃: SDK 300초 + 용어 10분 + 전체 30분
- 3단계 중복 체크: slug + exact term + 약어 매칭
- Exa deep context: Tavily < 3건일 때만 조건부 호출 (최신/희귀 용어 보강)

### 9. em dash 통일

프롬프트 예시 + skeleton 전체에서 `--` → `—` 통일. 뉴스 + 핸드북 모두.

## 핵심 배운 점

### 1. 팝업과 상세 페이지는 같은 필드로 서빙 가능
0_summary의 첫 줄을 전문 용어 없이 비유로 쓰면, `extractFirstSection`이 팝업용으로 자동 추출. 별도 필드 불필요.

### 2. "현상" 유형은 섹션 헤딩이 아니라 콘텐츠 관점을 바꿔야
섹션 키를 바꾸면 조립 코드가 깨짐. 프롬프트에서 "이 유형이면 관점을 바꿔라"로 가이드하는 게 유연.

### 3. 수식 렌더링은 도메인별 프로세서가 답
`singleDollarTextMath`를 전역으로 켜면 뉴스의 달러 표시가 깨짐. 프로세서를 분리하면 각 도메인에 맞는 파싱 가능.

### 4. few-shot 예시는 유형별로 다양하게
하나의 예시(Gemini 3.1)만 주면 LLM이 모든 용어를 "멀티모달 AI처럼" 설명. 5개 유형(개념/모델/도구/지표/현상)별 예시를 주면 유형에 맞는 콘텐츠 생성.

### 5. background task + DB 자동 저장은 세트
in-memory dict만으로는 polling 실패 시 결과 유실. DB 자동 저장을 함께 해야 안전.
