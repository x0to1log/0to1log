# 2026-03-25 비즈니스 피벗 결정

## 배경

프로젝트를 전반적으로 비즈니스 관점에서 냉철하게 검증함.
6가지 축으로 분석: 사용자 현실, 경쟁 환경, 기능 과부하, 수익화, 배포 채널, 액션 플랜.

### 주요 발견
- 사용자 0명 (Amy + 지인만)
- 사이트맵에 개별 콘텐츠 URL 0개 (SEO 치명적 버그)
- 배포 채널 0개 (뉴스레터, 소셜, 커뮤니티 모두 없음)
- 8개 테이블 행동 데이터 수집 중이나 분석 뷰 없음
- 기능 7개 동시 운영 — Solo로 감당 불가

## 결정 사항

### 1. 포지셔닝 확정
- **AI 용어/학습 도구** — AI를 처음 배우는 한국 개발자/기획자 대상
- **한국어 AI 뉴스** — 영어 뉴스를 직접 가져와서 한국어로 전달

### 2. 기능 우선순위 재정의
- **Core:** 핸드북(용어집) + AI 뉴스(한국어)
- **Keep:** 퀴즈, 라이브러리
- **Reshape:** AI 제품 → 뉴스 내 맥락적 도구 추천으로 전환
- **Deprioritize:** IT 블로그, 포트폴리오 (앱 전환 시 제거)

### 3. 수익화 전략 수정
- AdSense 제거 (학습 플랫폼에 광고 부적합)
- 순서: 뉴스레터 구독자 → 스폰서십 → 맥락적 affiliate → 프리미엄 구독
- 6개월+ 무수익 각오, 사용자 확보 우선

### 4. 배포 채널 확정
- 뉴스레터 (주간, Weekly Recap 파이프라인 기반)
- 긱뉴스, Reddit, X, Threads
- SEO (사이트맵 수정 후)

### 5. 다음 스프린트 최우선
- **Weekly Recap 파이프라인** → 뉴스레터 + 웹 + 커뮤니티 동시 배포의 기반

## 왜 이 결정을 했는가

- AI 뉴스 큐레이션은 레드오션 (The Rundown 400만+, TLDR 100만+)
- 한국어 시장은 비교적 비어있음. "학습 + 뉴스" 조합은 경쟁자에 없는 차별점
- Solo 프로젝트에서 7개 기능은 "1km wide, 1cm deep" → 핵심 2개에 집중
- "만들면 올 것이다"는 가장 흔한 실패 원인. 배포 채널이 먼저
