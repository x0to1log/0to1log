# Solo 프로젝트 비용 모델링

> Free Tier를 최대한 쓰고, KPI 게이트를 통과할 때만 돈을 낸다.

---

## 왜 지금 정리하나

News Pipeline v3 설계를 마무리하면서 일일 비용이 ~$0.59로 확정됐다. 월 단위로 환산하면 ~$18. 여기에 인프라 비용을 더하면 프로젝트 운영에 실제로 얼마가 드는지 구체적인 숫자가 나온다.

Solo 프로젝트에서 비용은 "얼마나 쓸 수 있느냐"가 아니라 **"언제 업그레이드를 정당화할 수 있느냐"** 의 문제다. 그 기준을 미리 정해두지 않으면 막연히 무료 한도를 넘기거나, 반대로 필요한 곳에 쓰지 못하고 성능을 포기하게 된다.

---

## 인프라 비용 현황

### Supabase — Free Tier 유지 중

| 항목 | 현재 | 한도 |
|---|---|---|
| DB 용량 | 측정 중 | 500MB |
| 12개월 후 예상 | ~75MB | — |

Free Tier로 1년은 충분히 버틸 수 있다. 다만 게이트를 미리 정해두지 않으면 어느 날 갑자기 한도에 가까워졌을 때 당황하게 된다.

**업그레이드 게이트:**

- **300MB** → `pipeline_logs` 오래된 실행 로그 아카이브
- **400MB** → rejected candidates 정리 (rejected 상태로 30일 이상 된 행 삭제)
- **450MB** → Supabase Pro 업그레이드 ($25/mo)

300MB 도달이 가장 빠른 트리거인데, 운영 로그가 하루에 수십 행씩 쌓이기 때문에 콘텐츠 자체보다 로그가 더 빨리 증가한다. 300MB 게이트를 아카이브 자동화로 연결해두는 게 맞다.

### Railway — $5/mo (Hobby Plan)

FastAPI 백엔드. cron 실행과 admin API 전용으로만 쓴다. 트래픽이 거의 없기 때문에 현재 플랜으로 충분하다. 부하가 생기는 시점은 외부 사용자가 실시간으로 생성 요청을 보낼 때인데, 그 전까지는 Hobby로 유지한다.

### Vercel — Free Tier

Astro 프론트엔드. Cron 트리거도 포함. 트래픽 기준 업그레이드 게이트는 월 100GB bandwidth 도달 시 Pro 검토 ($20/mo). 지금은 0에 가깝다.

### Tavily API — Free Tier

현재 뉴스 수집 규모(일 1회 실행, 카테고리별 쿼리 6~8개)는 Free Tier로 커버된다. 수집 빈도를 올리거나 카테고리를 추가할 경우 재검토 필요.

## LLM 비용

### 모델별 단가

| 모델 | Input | Output |
|---|---|---|
| gpt-4o | $2.50 / 1M tokens | $10.00 / 1M tokens |
| gpt-4o-mini | $0.15 / 1M tokens | $0.60 / 1M tokens |

gpt-4o-mini는 gpt-4o 대비 input 17배, output 17배 저렴하다. 분류/랭킹처럼 짧고 구조화된 작업은 gpt-4o-mini로 충분하다. 품질이 실제로 드러나는 콘텐츠 생성 단계에만 gpt-4o를 쓴다.

### Pipeline 비용 히스토리

| 버전 | 일일 비용 | 비고 |
|---|---|---|
| v1 | $0.42~1.26 | 재시도 포함. 대부분 실패 실행 |
| v2 | ~$0.43 | 안정적. 단일 뉴스 2건/일 |
| v3 | ~$0.59 | 다이제스트 3~5건/카테고리 |

v1에서 $1이 넘는 날이 있었던 건 파이프라인이 중간에 실패해도 앞단 LLM 호출 비용은 이미 소모됐기 때문이다. v2에서 draft 저장 + 스테이지 로깅을 도입하면서 재시도 낭비가 없어졌다. v3는 카테고리별 생성이 늘었지만 gpt-4o-mini 비중이 올라가서 단가가 억제됐다.

## 월 운영 비용 추산

| 항목 | 월 비용 |
|---|---|
| Pipeline (LLM) | ~$18 |
| Railway | $5 |
| Supabase | $0 (Free Tier) |
| Vercel | $0 (Free Tier) |
| Tavily | $0 (Free Tier) |
| **합계** | **~$23/mo** |

월 $23. 뉴스레터 구독 2개 정도의 금액으로 AI 뉴스 큐레이션 플랫폼을 운영하는 수준이다. Phase 1 MVP로는 허용 가능한 비용이다.

## 비용 추적 구조

모든 파이프라인 실행은 `pipeline_logs` 테이블에 기록된다.

추적 항목:
- 스테이지별 토큰 수 (input / output 분리)
- 스테이지별 비용 (USD)
- 실행 성공/실패 여부
- 재시도 횟수

이 데이터를 누적하면 두 가지를 할 수 있다:

1. **ROI 분석** — 뉴스 1건 생성에 실제로 얼마가 드는지, 시간이 지나면서 비용 구조가 어떻게 변하는지 추적
2. **이상 감지** — 특정 날 비용이 평균의 2배 이상이면 파이프라인 오작동 가능성 → [[Cost-Model-&-Stage-AB]] 에서 스테이지별 비용 벤치마크와 연결

## 업그레이드 의사결정 원칙

**KPI 게이트를 먼저 통과하고, 그 다음에 돈을 낸다.**

스케일링 비용을 먼저 내고 사용자를 기다리는 방식은 Solo 프로젝트에서 작동하지 않는다. 반대로, 무료 한도 안에서 실제 사용 패턴이 입증되면 업그레이드가 정당화된다.

구체적으로:

| 상황 | 조건 | 액션 |
|---|---|---|
| Supabase 용량 | 300MB 도달 | 로그 아카이브 자동화 |
| Supabase 용량 | 450MB 도달 | Pro 업그레이드 ($25/mo) |
| Railway 성능 | 응답 지연 > 10초 (admin 기준) | 플랜 검토 |
| Vercel bandwidth | 월 100GB | Pro 검토 ($20/mo) |
| LLM 비용 | 월 $30 초과 | 스테이지별 모델 재검토 |
| LLM 비용 | 월 $50 초과 | gpt-4o 사용 범위 축소 |

월 $50 LLM 게이트는 [[Monetization-Roadmap]] 의 수익 시점과 연결된다. 수익이 발생하기 전에 LLM 비용이 $50을 넘으면, 모델 전략을 다시 봐야 한다는 신호다.
