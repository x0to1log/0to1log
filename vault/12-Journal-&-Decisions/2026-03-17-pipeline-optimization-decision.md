# 결정: 파이프라인 실행 시간 최적화

> 날짜: 2026-03-17
> 맥락: 파이프라인 전체 구조 분석 후 병목 식별

---

## 문제

파이프라인 실행 시간 ~3분. 모든 LLM 호출이 순차 실행되고 있어 불필요한 대기 발생.

- Research 다이제스트 3 페르소나 완료 후에야 Business 시작 (서로 독립)
- 핸드북 4-call 중 Call 2(EN Basic)와 Call 3(KO Advanced)가 순차 (동시 실행 가능)
- 핸드북 용어 5개가 하나씩 순차 생성

## 결정

**병렬화로 실행 시간 47% 단축** (비용 변화 없음).

| 최적화 | 시간 절감 |
|--------|----------|
| Research + Business 다이제스트 동시 생성 | ~25s |
| 핸드북 Call 2 \|\| Call 3 병렬 | 용어당 ~5s |
| 핸드북 용어 2개씩 동시 생성 | ~30s |
| **합계** | **~80s (170s → 90s)** |

## 하지 않기로 한 것

- **분류(classify)에 gpt-4o-mini 사용**: 비용 절감 ~$0.03/일이지만 분류 품질 리스크. 데이터 충분히 쌓인 후 A/B 테스트.
- **퀄리티 체크 제거**: ~$0.004/일 절감이지만 자동 발행(Phase 3)의 선행 조건이므로 유지.

## 추가 발견

- `_generate_post()` 함수가 데드 코드 (v3 다이제스트 전환 후 미사용)
- `_filter_terms_with_llm()` 호출부 제거 완료, 함수 자체도 정리 대상
- 페르소나 부분 실패 시 포스트 저장하는 버그 발견 → 수정 완료 (3개 모두 성공 시에만 저장)
