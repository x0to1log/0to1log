# ADR: News Pipeline v2 → v3 전환 결정

> 날짜: 2026-03-16
> 상태: 결정됨
> 이전: [[2026-03-15-news-pipeline-v1-postmortem]]
> 설계: [[2026-03-16-daily-digest-design]]

---

## 맥락

News Pipeline v2는 하루에 **research 1건 + business 1건**을 선정하여 3 페르소나로 상세 분석하는 구조.

### v2의 문제점

1. **Research와 Business의 차이가 없음** — 둘 다 "1건 뉴스를 3 관점으로 분석". 선정 기준만 다르고 콘텐츠 형태는 동일.
2. **하루의 AI 동향을 파악할 수 없음** — 20건 수집해서 1건만 쓰고 나머지 버림. 독자 입장에서 "오늘 AI 업계에서 뭔 일이 있었지?"에 답을 못 함.
3. **다른 뉴스 플랫폼과 차별점 부족** — 단일 뉴스 분석은 어디서나 할 수 있음.

### 왜 지금 바꾸나

- v2 파이프라인 인프라가 안정화됨 (수집, 로깅, 비용 추적, 에러 처리, retry)
- 프론트엔드 페르소나 탭 전환 시스템 동작 중
- 핸드북 자동 추출 연결 완료
- 콘텐츠 형태를 바꿀 타이밍

## 결정

**v3: Daily Digest 형태로 전환**

### 핵심 변경

| | v2 | v3 |
|---|---|---|
| 형태 | 뉴스 1건 상세 분석 | 하루 3~5건 큐레이션 다이제스트 |
| Research | 기술 뉴스 1건 분석 | LLM & Models / Open Source / Papers 카테고리별 정리 |
| Business | 비즈니스 뉴스 1건 분석 | Big Tech / Industry & Biz / New Tools 카테고리별 정리 |
| 페르소나 | 같은 뉴스를 3번 다른 관점으로 | 같은 뉴스 목록, 설명의 깊이만 다르게 |
| 비용 | ~$0.43/일 | ~$0.59/일 (+37%) |

### 페르소나 차별화

- **Expert**: 벤치마크, 파라미터, 아키텍처, 코드 링크 포함. 리서치 엔지니어 브리핑 톤.
- **Learner**: 기술 배경 + 왜 중요한지 + 실무 영향. 개발자 뉴스레터 톤.
- **Beginner**: 비유로 설명 + 일상 영향 + 핸드북 링크. 쉬운 AI 뉴스 톤.

### 구현 계획

4 Phase로 진행:
1. DIGEST-01: 랭킹 → 카테고리별 분류
2. DIGEST-02: 다이제스트 프롬프트
3. DIGEST-03: 파이프라인 오케스트레이터
4. DIGEST-04: 프론트엔드 렌더링

---

## 고려한 대안

### A. v2 유지 + 뉴스 건수만 늘리기
- 1건 → 3건으로 늘리되 각각 독립 포스트로 발행
- 장점: 변경 최소. 단점: 여전히 개별 분석이라 다이제스트 느낌 없음
- **기각**: 근본적 차별화 안 됨

### B. 다이제스트 + 개별 심층 분석 병행
- 다이제스트 1개 + 가장 중요한 뉴스 1건은 v2 스타일 상세 분석
- 장점: 둘 다 제공. 단점: 비용 2배, 운영 복잡
- **보류**: v3 안정화 후 재검토

---

## 영향

- **DB**: `news_posts` 테이블 구조 변경 불필요 (content 필드에 다이제스트 전체가 들어감)
- **프론트엔드**: 뉴스 상세 페이지 렌더링 방식 변경 필요
- **핸드북 추출**: 다이제스트에 포함된 여러 뉴스에서 용어 추출 → 더 많은 용어 자동 생성
- **비용**: 일일 ~$0.16 증가 (월 ~$5)
