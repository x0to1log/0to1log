## ProductCard 구조

```
┌────────────────────────────────┐
│  [Logo]  이름           [무료] │  ← 가격 배지
│          tagline 한 줄         │
│          ─────────────────     │
│          🌐 💻 📱  [한국어 ✓] │  ← 플랫폼 + 한국어 지원
└────────────────────────────────┘
  클릭 → /en/products/[slug]/
  hover → 팝오버 (tagline + 방문 버튼)
```

### 가격 배지 색상
- `free` → 초록
- `freemium` → 파랑
- `paid` → 주황
- `enterprise` → 보라

## 반응형 그리드

| 뷰포트 | ProductCard 열 수 |
|---|---|
| 데스크탑 (≥1024px) | 3열 |
| 태블릿 (≥768px) | 2열 |
| 모바일 (<768px) | 1열 |

Hero featured 카드:
| 뷰포트 | 레이아웃 |
|---|---|
| 데스크탑 | 가로 배치 (최대 5개) |
| 모바일 | 세로 스택 (최대 3개) |

## UX 원칙

1. **에디토리얼 카피 우선**: 기술 설명 전에 "이걸로 뭘 할 수 있는지" 먼저
2. **배지로 빠른 스캔**: 가격/플랫폼/한국어 지원을 한눈에 파악
3. **미디어 먼저**: 상세 페이지에서 글 전에 영상/이미지 먼저
4. **초보자 언어**: "프롬프트" 대신 "텍스트 입력", "inference" 대신 "생성"
5. **발견의 감동**: Hero + 카테고리 에디토리얼이 "이런 도구들이 있었어?" 감각 유발
