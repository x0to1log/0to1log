# Brand Identity

0to1log의 브랜드 정체성. 디자인 시스템([[Design-System]])이 "어떻게 보이는가"를 정의한다면, 이 문서는 "어떤 존재인가"를 정의한다.

---

## 브랜드 네임

**0to1log** — 이름 하나에 4가지 레이어가 담겨 있다.

### 4가지 숨은 의미

| 레이어 | 의미 | 설명 |
|---|---|---|
| **무에서 유로의 창조** | Zero to One | 피터 틸의 《Zero to One》에서 유래. 기존의 것을 복제(1→n)하는 것이 아니라 **새로운 가치를 만들어내는 과정**. AI 모델을 처음부터 설계하고 구현하는 엔지니어의 숙명 |
| **AI의 근간, 이진법** | Binary World | 컴퓨터와 AI의 가장 기초적인 단위 **0과 1**. "세상의 모든 데이터는 0과 1로 이루어져 있다"는 기술적 본질 |
| **성장의 비선형성** | Logarithmic Growth | log 함수는 초반에 급격히 상승하다가 점차 안정적인 궤도에 오름. 폭발적인 학습을 거쳐 **전문성을 갖춘 엔지니어로 거듭나는 성장 곡선** |
| **연결의 기록** | The Connectivity | 0(시작점)과 1(목표점) 사이를 잇는 'to'는 그 자체로 **과정**. 결과만 보여주는 포트폴리오가 아니라, 고군분투하는 모든 과정을 기록(log)하겠다는 진정성 |

## 브랜드 보이스 & 톤

5가지 보이스 키워드:

| 키워드 | 설명 | 예시 |
|---|---|---|
| **명확한** | 기술 용어를 쓰되, 초보자도 이해할 수 있게 설명 | "LLM이 텍스트를 생성합니다" → "AI가 글을 써줍니다" (입문자 버전) |
| **신뢰할 수 있는** | 출처를 밝히고, 불확실한 건 불확실하다고 표시 | "~로 알려져 있다" / "공식 발표에 따르면" |
| **과장 없는** | AI 뉴스의 과대포장(hype)을 지양 | "혁명적" → "주목할 만한 변화" |
| **도움되는** | 읽고 나서 "그래서 나한테 뭐가 달라지는데?"에 답할 수 있어야 함 | 기술 설명 후 반드시 실용적 시사점 포함 |
| **친절한** | 모르는 걸 부끄러워하지 않게. 질문을 환영하는 태도 | "쉽게 말하면 ~", "처음이라면 ~부터 시작해보세요" |

### 페르소나별 톤 차이

| 페르소나 | 톤 |
|---|---|
| **입문자 (Beginner)** | 친절하고 설명적. 비유와 일상 예시 활용 |
| **현직자 (Practitioner)** | 간결하고 밀도 높음. 기술 용어 그대로 사용 |

## 브랜드 스토리

포트폴리오/면접에서 사용할 수 있는 내러티브 구조:

| 단계 | 내용 |
|---|---|
| **문제 인식** | AI 뉴스는 영어 중심이고, 초보자에게 너무 어렵거나 과대포장되어 있다. 한국어로 원문 품질을 유지하면서 수준별로 전달하는 서비스가 없다 |
| **해결 방법** | LLM 에이전트로 매일 자동 큐레이션 + 페르소나별 난이도 조절. 뉴스 + 용어집 + AI 제품을 통합한 학습형 플랫폼 |
| **Solo 도전** | 1인 개발자가 Claude Code와 함께 기획 → 설계 → 구현 → 배포 → 운영까지 전체 사이클을 담당 |
| **이름의 의미** | 0to1log = 무에서 유를 만드는 과정의 기록. 결과가 아닌 과정을 투명하게 공유하는 것이 이 프로젝트의 진정성 |
| **성장** | 데이터 기반 의사결정(KPI Gate)으로 사이드 프로젝트를 지속 가능한 프로덕트로 발전시키는 과정 |
